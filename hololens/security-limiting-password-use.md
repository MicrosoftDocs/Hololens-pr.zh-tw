# <span data-ttu-id="4be3e-104">減少使用密碼</span><span class="sxs-lookup"><span data-stu-id="4be3e-104">Limiting password use</span></span>

<span data-ttu-id="4be3e-105">現今大多數的電腦系統都是利用使用者認證作為安全性的基礎，以致這些系統依賴這些可重複使用、由使用者建立的密碼。</span><span class="sxs-lookup"><span data-stu-id="4be3e-105">Most computer systems today utilize user credentials as the basis for security making them dependent on reusable, user-created passwords.</span></span> <span data-ttu-id="4be3e-106">這也導致密碼成為帳戶入侵和資料外洩的最常見原因。</span><span class="sxs-lookup"><span data-stu-id="4be3e-106">This has resulted in passwords also becoming the most common cause of account compromise and data breaches.</span></span> <span data-ttu-id="4be3e-107">舉例來說，密碼會在傳輸過程中遭到攔截或是從伺服器竊取而得 (透過網路釣魚或密碼噴濺攻擊)，然後盜用以取得使用者帳戶的存取權。</span><span class="sxs-lookup"><span data-stu-id="4be3e-107">As an example of this, passwords can be intercepted in transmission or stolen from a server (by phishing or password spray attacks) and compromised to gain access to a user account.</span></span>

<span data-ttu-id="4be3e-108">為了改善安全性和帳戶防護，HoloLens 2 具備有功能可以針對裝置的登入啟用增強式，即硬體支援的「無密碼」認證 (包括 Windows Hello)，讓您存取 Microsoft 雲端時順暢無阻。</span><span class="sxs-lookup"><span data-stu-id="4be3e-108">To improve security and account protection, HoloLens 2 has the capability to enable strong, hardware-backed “password-less” credentials (including Windows Hello) for device sign in, offering seamless access to the Microsoft cloud.</span></span> 

## <span data-ttu-id="4be3e-109">從另一部裝置登入</span><span class="sxs-lookup"><span data-stu-id="4be3e-109">Signing in from another device</span></span>

<span data-ttu-id="4be3e-110">在初始裝置設定和使用者登入期間，HoloLens 2 會針對 Azure Active Directory 工作帳戶提供遠端裝置登入的選項，降低輸入複雜密碼的必要性，以及最小化將密碼當成認證的必要性。</span><span class="sxs-lookup"><span data-stu-id="4be3e-110">HoloLens 2 offers remote device sign-in options for Azure Active Directory work accounts during initial device setup and user sign-in to reduce the need to type complex passwords and minimize the need for passwords as credentials.</span></span> <span data-ttu-id="4be3e-111">使用智慧卡進行驗證的使用者和組織很難在 HoloLens 2 之類的裝置上使用這些認證，而且組織經常會開發出複雜的系統和高成本的程序來解決這個問題。</span><span class="sxs-lookup"><span data-stu-id="4be3e-111">Users and organizations who use smartcards to authenticate have difficulty using those credentials on devices such as HoloLens 2, and often organizations develop complicated systems and costly processes to work around the problem.</span></span> <span data-ttu-id="4be3e-112">為了解決這個問題，Azure AD 提供兩個在 HoloLens 2 上無密碼登入的選項。</span><span class="sxs-lookup"><span data-stu-id="4be3e-112">To solve this problem, Azure AD offers two options for password-less sign-in on HoloLens 2.</span></span> 

<span data-ttu-id="4be3e-113">第一個驗證方法仰賴 Microsoft Authenticator 應用程式中的新功能，提供可以將使用者認證繫結到裝置的金鑰型驗證。</span><span class="sxs-lookup"><span data-stu-id="4be3e-113">The first authentication method relies on new capabilities in the Microsoft Authenticator app to provide key-based authentication that enables a user credential tied to a device.</span></span> <span data-ttu-id="4be3e-114">系統管理員針對租用戶啟用這項功能之後，系統會在 HoloLens 裝置設定期間向使用者顯示訊息，通知使用者在其應用程式上點選數字。</span><span class="sxs-lookup"><span data-stu-id="4be3e-114">Once enabled on a tenant by the administrator, users will be shown a message during HoloLens device setup telling them to tap a number on their app.</span></span> <span data-ttu-id="4be3e-115">接著，使用者比對驗證器應用程式中的數字、選擇 [核准]、提供 PIN 或生物特徵辨識，然後完成驗證，使 HoloLens 設定繼續進行。</span><span class="sxs-lookup"><span data-stu-id="4be3e-115">They must then match the number on their authenticator app, choose Approve, provide their PIN or a biometric and complete authentication for their HoloLens setup to proceed.</span></span> <span data-ttu-id="4be3e-116">這部分在[無密碼登入](https://docs.microsoft.com/azure/active-directory/authentication/howto-authentication-passwordless-phone)中有更詳細的說明。</span><span class="sxs-lookup"><span data-stu-id="4be3e-116">This is described in greater detail in [passwordless sign-in](https://docs.microsoft.com/azure/active-directory/authentication/howto-authentication-passwordless-phone).</span></span>

<span data-ttu-id="4be3e-117">第二個選項是裝置代碼流程，對使用者而言是直覺易用，而且不需要任何其他的基礎結構。</span><span class="sxs-lookup"><span data-stu-id="4be3e-117">The second is a device code flow that is intuitive to users and does not require any additional infrastructure.</span></span>  <span data-ttu-id="4be3e-118">這個遠端登入行為仰賴另一部受信任的裝置，該裝置支援組織慣用的驗證機制，並且會在完成時發出權杖傳回 HoloLens 以完成登入或裝置設定。</span><span class="sxs-lookup"><span data-stu-id="4be3e-118">This remote sign-in behavior relies on another trusted device that supports the organization’s preferred authentication mechanism and when complete, tokens are issued back to the HoloLens to complete sign-in or device setup.</span></span> <span data-ttu-id="4be3e-119">這個流程中的步驟如下：</span><span class="sxs-lookup"><span data-stu-id="4be3e-119">The steps in this flow are:</span></span>

  1.    <span data-ttu-id="4be3e-120">在 OOBE (全新體驗) 時完成初始裝置設定或登入流程的使用者會看到 [從另一部裝置登入] 連結，點選該連結。</span><span class="sxs-lookup"><span data-stu-id="4be3e-120">A user going through the initial device setup or sign-in flows on OOBE is presented with a “Sign in from another device” link and taps on it.</span></span> <span data-ttu-id="4be3e-121">這會啟動遠端登入工作階段。</span><span class="sxs-lookup"><span data-stu-id="4be3e-121">This initiates a remote sign in session.</span></span>
  2.    <span data-ttu-id="4be3e-122">接著，系統會向使用者顯示包含有簡短 URI ([https://microsoft.com/devicelogin](https://microsoft.com/devicelogin)) 的輪詢頁面，指向 Azure AD Secure 安全性權杖服務 (STS) 的裝置驗證端點。</span><span class="sxs-lookup"><span data-stu-id="4be3e-122">The user is then shown a polling page which contains a short URI ([https://microsoft.com/devicelogin](https://microsoft.com/devicelogin)) that points to the device authentication endpoint of the Azure AD Secure Token Service (STS).</span></span> <span data-ttu-id="4be3e-123">使用者也會看到一組一次性代碼，此代碼在雲端中安全地產生，最長存留期為15分鐘。</span><span class="sxs-lookup"><span data-stu-id="4be3e-123">The user is also presented with a one-time code that is securely generated in the cloud and has a maximum lifetime of 15 minutes.</span></span> <span data-ttu-id="4be3e-124">隨著代碼的產生，Azure AD 在先前步驟中初始化遠端登入要求時，也會建立加密的工作階段，然後一併使用 URI 和代碼來批准遠端登入要求。</span><span class="sxs-lookup"><span data-stu-id="4be3e-124">Along with code generation, Azure AD also creates an encrypted session upon initiation of the remote sign in request in the previous step and together, the URI and code are used to approve the remote sign-in request.</span></span> 
  3.    <span data-ttu-id="4be3e-125">使用者接著在另一部裝置上瀏覽到該 URI，系統會提示使用者輸入 HoloLens 2 裝置上顯示的代碼。</span><span class="sxs-lookup"><span data-stu-id="4be3e-125">The user then navigates to the URI from another device and is prompted to enter the code displayed on their HoloLens 2 device.</span></span> 
  4.    <span data-ttu-id="4be3e-126">使用者輸入代碼後，Azure AD STS 會顯示頁面，指出使用者的 HoloLens 2 裝置觸發遠端登入要求並要求產生代碼。</span><span class="sxs-lookup"><span data-stu-id="4be3e-126">Once the user enters the code, Azure AD STS displays a page indicating the user’s HoloLens 2 device that triggered the remote sign in request and requested generation of the code.</span></span> <span data-ttu-id="4be3e-127">系統接著提示使用者確認，避免任何的網路釣魚攻擊。</span><span class="sxs-lookup"><span data-stu-id="4be3e-127">The user is then prompted for confirmation to prevent any phishing attacks.</span></span> 
  5.    <span data-ttu-id="4be3e-128">如果使用者選擇繼續登入顯示的「應用程式」，Azure AD STS 會提示使用者輸入其認證。</span><span class="sxs-lookup"><span data-stu-id="4be3e-128">If the user chooses to continue signing in to the displayed ‘application’, Azure AD STS prompts the user for their credentials.</span></span> <span data-ttu-id="4be3e-129">驗證成功時，Azure AD STS 會將快取的遠端工作階段更新為「已核准」並隨附一組授權碼。</span><span class="sxs-lookup"><span data-stu-id="4be3e-129">On successful authentication, Azure AD STS updates the cached remote session as 'approved' along with an authorization code.</span></span>
  6.    <span data-ttu-id="4be3e-130">最後，使用者 HoloLens 2 裝置上的輪詢頁面會收到來自 Azure AD 的「已授權」回應，然後繼續驗證使用者代碼、其相關聯的已儲存授權碼，並依照要求產生 OAuth 權杖以完成裝置設定。</span><span class="sxs-lookup"><span data-stu-id="4be3e-130">Finally, the polling page on the user’s HoloLens 2 device receives an 'Authorized' response from Azure AD and proceeds to validate the user code, its associated stored authorization code and generates OAuth tokens as requested to complete device setup.</span></span> <span data-ttu-id="4be3e-131">建立的驗證權杖有效時間為 1 小時，重新整理權杖則有 90 天的存留期。</span><span class="sxs-lookup"><span data-stu-id="4be3e-131">The authentication token created is valid for 1 hour and the refresh token has a lifetime of 90 days.</span></span> 

<span data-ttu-id="4be3e-132">這個流程中使用的代碼產生和加密演算法皆符合 FIPS 規範。</span><span class="sxs-lookup"><span data-stu-id="4be3e-132">The code generation and encryption algorithms used in this flow are both FIPS compliant.</span></span> <span data-ttu-id="4be3e-133">HoloLens 2 裝置利用 TPM 保護裝置金鑰的安全，並使用硬體保護金鑰加密在使用者驗證之後產生的權杖。</span><span class="sxs-lookup"><span data-stu-id="4be3e-133">HoloLens 2 devices utilize the TPM to secure device keys and encrypt tokens generated after user authentication using hardware-protected keys.</span></span> <span data-ttu-id="4be3e-134">有關 HoloLens 2 中權杖安全性的詳細資訊已於[主要重新整理權杖 (PRT)](https://docs.microsoft.com/azure/active-directory/devices/concept-primary-refresh-token) 中分享。</span><span class="sxs-lookup"><span data-stu-id="4be3e-134">More information on token security on HoloLens 2 is shared in [What is a Primary Refresh Token (PRT)](https://docs.microsoft.com/azure/active-directory/devices/concept-primary-refresh-token).</span></span>

## <span data-ttu-id="4be3e-135">使用 Windows Hello 登入裝置</span><span class="sxs-lookup"><span data-stu-id="4be3e-135">Device sign-in with Windows Hello</span></span>

<span data-ttu-id="4be3e-136">[Windows Hello](https://docs.microsoft.com/windows/security/identity-protection/hello-for-business/hello-identity-verification) 提供的無密碼選項直接內建在作業系統中，讓使用者利用虹膜或 PIN 就能登入裝置。</span><span class="sxs-lookup"><span data-stu-id="4be3e-136">[Windows Hello](https://docs.microsoft.com/windows/security/identity-protection/hello-for-business/hello-identity-verification) offers password-free options built directly into the operating system permitting users to sign into the device using Iris or PIN.</span></span> <span data-ttu-id="4be3e-137">PIN 始終可以提供作為認證而且是裝置設定的必要條件，虹膜則是選用項目可以略過。</span><span class="sxs-lookup"><span data-stu-id="4be3e-137">PIN is always available as a credential and is required for device setup, whereas Iris is optional and may be skipped.</span></span> <span data-ttu-id="4be3e-138">使用者可以使用個人的 Microsoft 帳戶或 [Azure Active Directory 工作帳戶登入 HoloLens 裝置，而*不必*輸入密碼](https://docs.microsoft.com/azure/active-directory/authentication/concept-authentication-passwordless)。</span><span class="sxs-lookup"><span data-stu-id="4be3e-138">Users can sign-in to HoloLens devices using their personal Microsoft account or [Azure Active Directory work account *without* entering a password](https://docs.microsoft.com/azure/active-directory/authentication/concept-authentication-passwordless).</span></span> <span data-ttu-id="4be3e-139">這類選項可讓使用者快速、安全地存取完整的 Windows 體驗、應用程式、資料、網站和服務。</span><span class="sxs-lookup"><span data-stu-id="4be3e-139">Options such as these offer users rapid, secured access to their full Windows experience, apps, data, websites, and services.</span></span> <span data-ttu-id="4be3e-140">此處詳細說明 Microsoft 朝向無密碼體驗的策略。</span><span class="sxs-lookup"><span data-stu-id="4be3e-140">Microsoft’s strategy towards passwordless experiences is detailed here.</span></span>

<span data-ttu-id="4be3e-141">建立 Windows Hello 認證時，認證會與身分識別提供者建立受信任的關係，並建立一個用來驗證的非對稱金鑰組。</span><span class="sxs-lookup"><span data-stu-id="4be3e-141">When a Windows Hello credential is created, it establishes a trusted relationship with an identity provider and creates an asymmetric key-pair for authentication.</span></span> <span data-ttu-id="4be3e-142">Windows Hello 手勢 (例如虹膜或 PIN) 提供熵，用來解密裝置的信賴平台模組 (TPM) 晶片支援的私密金鑰。</span><span class="sxs-lookup"><span data-stu-id="4be3e-142">A Windows Hello gesture (such as iris or PIN) provides entropy to decrypt a private key backed by the device’s Trusted Platform Module (TPM) chip.</span></span> <span data-ttu-id="4be3e-143">這個私密金鑰接著會用於簽署傳送到驗證伺服器的要求，在驗證成功之後，使用者會被授與其郵件、圖片及其他帳戶設定的存取權。</span><span class="sxs-lookup"><span data-stu-id="4be3e-143">This private key is then used to sign requests sent to an authenticating server and upon successful authentication, the user is granted access to their mail, pictures, and other account settings.</span></span> 

<span data-ttu-id="4be3e-144">如需相關資訊，請參閱下列資訊圖表：</span><span class="sxs-lookup"><span data-stu-id="4be3e-144">For more information, see the following infographic:</span></span>

  ![Windows Hello 登入](images/security-hello-sign-in.png)
  
<span data-ttu-id="4be3e-146">請注意，在上面顯示的圖形中，nonce 代表「數字一次」，而且是隨機或半隨機產生的數字。</span><span class="sxs-lookup"><span data-stu-id="4be3e-146">In the graphic presented above, note that nonce is stands for “number once”, and is a random or semi-random generated number.</span></span> <span data-ttu-id="4be3e-147">Windows Hello 生物特徵辨識或 PIN 認證設定完成之後，永遠不會離開佈建這些認證所在的裝置。</span><span class="sxs-lookup"><span data-stu-id="4be3e-147">Once the Windows Hello Biometric or PIN credential is set up, it never leaves the device on which it is provisioned.</span></span> <span data-ttu-id="4be3e-148">即使使用者的 Windows Hello PIN 遭竊，例如透過網路釣魚攻擊，該認證[在沒有使用者實體裝置的情況之下毫無用處](https://docs.microsoft.com/windows/security/identity-protection/hello-for-business/hello-why-pin-is-better-than-password)。</span><span class="sxs-lookup"><span data-stu-id="4be3e-148">Even if the user’s Windows Hello PIN is stolen, such as through a phishing attack, it is [useless without the user’s physical device](https://docs.microsoft.com/windows/security/identity-protection/hello-for-business/hello-why-pin-is-better-than-password).</span></span>

<span data-ttu-id="4be3e-151">虹膜驗證回復為 PIN。</span><span class="sxs-lookup"><span data-stu-id="4be3e-151">Iris authentication falls back to the PIN.</span></span> <span data-ttu-id="4be3e-152">為了在新裝置上設定新 PIN (增強式驗證器)，使用者必須在最近完成[多重要素驗證 (MFA)](https://docs.microsoft.com/azure/active-directory/authentication/concept-mfa-howitworks) 才能完成程序。</span><span class="sxs-lookup"><span data-stu-id="4be3e-152">In order to set up a new PIN (a strong authenticator) on the device, the user must have recently gone through [Multifactor Authentication (MFA)](https://docs.microsoft.com/azure/active-directory/authentication/concept-mfa-howitworks) in order to complete the process.</span></span>
